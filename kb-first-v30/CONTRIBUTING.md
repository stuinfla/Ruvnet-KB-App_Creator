# Contributing to KB-First

Thank you for your interest in contributing to KB-First!

## Ways to Contribute

### 1. Report Issues
- Bug reports
- Feature requests
- Documentation improvements

### 2. Submit Pull Requests
- Bug fixes
- New features
- Documentation updates
- New pattern guides
- Additional templates

## Guidelines

### Code Style
- TypeScript for all code
- Clear comments explaining KB enforcement
- Follow existing patterns

### Documentation
- Keep markdown clean and readable
- Include code examples
- Update README if adding features

### Testing
- Verify KB enforcement rules pass
- Test with actual PostgreSQL instance
- Include example queries

## Process

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Run verification: `./scripts/verify-enforcement.sh`
5. Submit a pull request

## Questions?

Open an issue for discussion.
